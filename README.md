# BAND ROOM

## アプリの概要とキャッチコピー
**「バンド活動を、もっとスムーズに。」**
スケジュール管理、セットリスト（音源）の共有、メンバー間のチャットをひとまとめにした、バンド専用のグループウェアアプリです。

## 開発背景・解決したい課題
私自身、普段からエレキギターを弾いており、マルチエフェクターでの緻密な音作りや、ハイゲインなアンプのサウンドを探求するなど、音楽活動に強い情熱を持っています。
しかし、いざバンドで集まるとなると「次のスタジオはいつだっけ？」「新曲のデモ音源、LINEの期限が切れて聞けない…」といった連絡やデータ管理が複数のアプリに分散し、非常に煩雑でした。

「もっと純粋に、演奏や音作りに集中できる環境を作りたい」
「プログラミング初心者でも、自分たちの課題を解決できるツールを作れるはずだ」

という思いから、ネットの資料やAIを活用しながら手探りでFlutterを学び、このアプリを開発しました。非同期処理や状態管理など、何度も壁にぶつかりましたが、1つずつ実装を重ねて形にすることができました。

## 主な機能一覧
* **ユーザー＆グループ認証 (Firebase Auth / Firestore)**
  * アカウント作成およびログイン機能。
  * 招待コードを用いたバンド（グループ）の作成・参加機能。これによりメンバー間だけでデータを安全に共有。
* **セットリスト・音源管理 (Firebase Storage / audioplayers)**
  * スマホ内の音源データ（MP3, WAV等）のアップロードとクラウド保存。
  * アプリ内での直接再生、曲の長さの自動検出、リスト総再生時間の自動計算。
  * 任意でオン/オフ可能な「自動連続再生」機能。
* **スケジュール管理 (table_calendar)**
  * カレンダー形式での直感的な予定（LiveやStudio等）の登録。
  * 日付タップ時のポップアップ表示と、予定の削除機能。
* **ルームチャット機能**
  * グループメンバー内でのリアルタイムなテキストメッセージのやり取り。

## 使用技術と技術選定の理由
* **フロントエンド: Flutter (Dart)**
  * **選定理由:** 1つのコードベースでiOSとAndroidの両方に展開できる点に魅力を感じました。また、豊富なWidgetを活用することで、白を基調とした洗練されたUI（影を使ったタブの選択状態など）を、初心者でも直感的に構築しやすかったためです。
* **バックエンド (BaaS): Firebase**
  * **Authentication:** セキュアなログインとグループ連携の基盤として使用。
  * **Cloud Firestore:** チャットのリアルタイム更新や、スケジュールの保存に最適だと判断しました。NoSQLの構造を理解するのに苦労しましたが、ドキュメントIDを使ったデータの削除機能などを実装できました。
  * **Cloud Storage:** バンドの大切な音源データを保存し、ストリーミング再生に繋げるために採用しました。
* **主なパッケージ**
  * `audioplayers`: 連続再生機能や、曲の長さ取得など、複雑なオーディオ制御を実現するため。
  * `table_calendar`: カレンダーUIをゼロから作るのではなく、既存の強力なライブラリを拡張して使う実践的な開発を学ぶため。

## ローカルでの動かし方（セットアップ手順）
このリポジトリをクローンしてローカルで動かす場合の手順です。

1. **Flutter環境の構築**
   Flutter SDKがインストールされていることを確認してください。
2. **リポジトリのクローン**
   ```bash
   git clone [このリポジトリのURL]
   cd [プロジェクトフォルダ名]
3. **パッケージのインストール**
   Bash

    flutter pub get
4. **Firebaseの設定（※重要）**
   本プロジェクトはFirebaseと連携しています。ローカルで実行する場合は、ご自身のFirebaseプロジェクトを作成し、firebase_options.dart を生成して配置する必要があります。

5. **アプリの実行**
   Bash

   flutter run
6. **インフラ設定について（Firebase Storage CORS設定）**
   Webブラウザ等でクラウド上の音源を再生する際、CORS（Cross-Origin Resource Sharing）制約によりエラーが発生する場合があります。
   Firebase Storageに対して、以下の設定（cors.json）を適用してください。

   cors.json

   JSON

   [
     {
       "origin": ["*"],
       "method": ["GET", "HEAD"],
       "responseHeader": ["Content-Type"],
       "maxAgeSeconds": 3600
     }
   ]
7. **適用コマンド（Google Cloud CLIを使用）**
   Bash

   gsutil cors set cors.json gs://[あなたのFirebase Storageのバケット名]
